# 시노부

시노부는 스트림릿 기반의 실전 자동매매 워크스페이스입니다.

현재 앱은 `KODEX 레버리지(122630)`와 `KODEX 200선물인버스2X(252670)`를 한 쌍으로 보고,
`5분봉` 기준 SCR 신호로 실제 매매를 수행하는 데 맞춰져 있습니다.

## 핵심 기능

- 실전 계좌 잔고 조회
- 실전 보유 포지션 조회
- `5분봉` 기준 자동 매매 실행 / 중지
- 레버리지 / 인버스 베타 스위칭
- 실전 매매 로그
- 실전 주문 이력 차트
- 한국시간 기준 차트 표시

## 현재 전략

- 진입: `buy open`
- 청산: `buy close`
- 추가 청산:
  - 반대 ETF에서 `buy open`이 뜨면 현재 ETF를 즉시 청산
  - 이후 반대 ETF로 바로 스위칭

즉,

- 레버리지를 들고 있을 때 인버스 `buy open`이 뜨면 레버리지 즉시 매도 후 인버스 매수
- 인버스를 들고 있을 때 레버리지 `buy open`이 뜨면 인버스 즉시 매도 후 레버리지 매수

## 실전 주문 기준

- 실전 주문은 `5분봉`에서만 동작합니다.
- 앱이 열려 있는 동안 `30초마다` 최신 완료 봉을 확인합니다.
- 최대 주문 투입금은 `500,000원`입니다.
- 주문은 한투 Open API를 사용합니다.

## 데이터 소스

- 국내주식 일봉 / 주봉 / 월봉: 한국투자 Open API 우선
- 국내주식 분봉: 한국투자 Open API 분봉 이력 기반
- 차트 시간 기준: `Asia/Seoul`

## 실행 방법

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 필수 시크릿

`.streamlit/secrets.toml` 예시:

```toml
KIS_APP_KEY = "..."
KIS_APP_SECRET = "..."
KIS_CANO = "12345678"
KIS_ACNT_PRDT_CD = "01"
KIS_IS_REAL = "true"
```

## 보안

아래 파일은 Git에 올라가지 않도록 처리되어 있습니다.

- `.streamlit/secrets.toml`
- `.env`

실전 계좌 키와 계좌번호는 반드시 시크릿 파일로만 관리해야 합니다.

- 왼쪽:
  - 실전 매매 차트
  - 실전 주문 이력
- 오른쪽:
  - 실계좌 잔고 / 보유 종목
  - 실전 투자 실행 / 중지
  - 실전 매매 로그

## 주의

이 프로젝트는 실전 주문 기능이 연결되어 있습니다.
버튼을 누르면 실제 주문이 나갈 수 있으므로, 실계좌 사용 시 반드시 계좌 상태와 전략 조건을 확인한 뒤 사용해야 합니다.
